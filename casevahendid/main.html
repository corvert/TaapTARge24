Mis on CASE-vahendid?

CASE (Computer Aided Software Engineering) vahendid on tarkvaraprogrammid, mis toetavad tarkvara arendus- ja haldusprotsessi. Need vahendid aitavad automatiseerida, hallata ja lihtsustada erinevaid arendusprotsessi etappe, sealhulgas nõuete analüüsi, arendustöö ajakava koostamist ja mudelite loomist.

CASE-vahendite klassifikatsioon

CASE-vahendeid jaotatakse nende konkreetse otstarbe järgi kahte põhikategooriasse:

Upper CASE Tools

Ülemise taseme vahendid toetavad analüüsi ja projekteerimist. Peamiselt kasutusel kasutajanõuete analüüsimisel, visualiseerimisel ja dokumenteerimisel.

Lower CASE Tools

Alumise taseme vahendid keskenduvad teostusele, kus mudelitest saab tegelik tarkvaratoode. Toetavad koodi genereerimist ja testimist.

CASE-vahendid spiraalmudeli etappides

1. Planeerimine

Nõuete kogumise tööriistad:

  • JIRA - projekti nõuete dokumenteerimine
  • Confluence - nõuete haldus

Ajakava koostamine:

  • Microsoft Project
  • Asana - ressursside haldus

2. Riskianalüüs

Riskihalduse tööriistad:

  • RiskWatch - riskide tuvastamine
  • RiskyProject - riskide jälgimine

Prototüüpimine:

  • Axure - prototüüpide loomine
  • Balsamiq - lahenduste visualiseerimine

3. Inseneritöö

Arendustööriistad:

  • Visual Studio - koodiredaktor
  • Eclipse - Java arendus

Testimisvahendid:

  • Selenium - automaattestimine
  • JUnit - ühiktestid

4. Hindamine

Tagasiside kogumine:

  • SurveyMonkey - küsitlused
  • Google Forms - andmete kogumine

Analüüsivahendid:

  • Google Analytics - kasutusstatistika
  • Mixpanel - kasutajate käitumine
